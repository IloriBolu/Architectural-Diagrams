# URL SHORTNER
## SCENARIO:
A startup wants a URL shortener like bit.ly. Users paste a long URL, get a short one back. When someone visits the short URL, they get sent to the original long URL.


## GOAL:
Design a serverless architecture that creates and redirects short URLs.


### DIAGRAM INCLUDES:
-	The API endpoint that creates a short URL
-	Where the mapping between short and long URLs is stored
-	How the redirect works when someone visits a short URL
-	The HTTP status codes involved (think about 301 vs 302)
