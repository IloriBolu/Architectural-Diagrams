## SCENARIO:
A team messaging app where users can send messages in channels and see messages appear instantly. Messages must be stored so users can scroll back through history.

## GOAL:
Design an architecture that supports real-time messaging with message persistence.

### DIAGRAM INCLUDES:
-	How users connect for real-time updates (WebSocket, not polling)
-	How a sent message reaches other users in the same channel
-	Where messages are stored for history
-	How users are authenticated
